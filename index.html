Linux

IP Address

Assigner une IP à une interface 
ip addr add [IP]/[MASK] dev [INAME]
ip addr add 192.168.1.1/24 dev eth0
Pour supprimer l'ip de l'interface, remplacez "add" par "del" dans la commande en haut.

Up une interface 
ip link set [INAME] up
ip link set eth0 up
Pour down l'interface, remplacez "up" par "down"

Interfaces

interface [INAME]
    duplex full
    speed [SPEED (10|100|1000)]
    mdix auto
interface Fa0/0/0
    duplex full
    speed 100
    mdix auto

Routes

Mettre en place une route 
ip route add [DST_NETWORK]/[DST_MASK] via [ROUTER_IP]
ip route add 172.16.0.0/16 via 192.168.0.254
Mettre en place la route par défaut 
ip route add default via [GATEWAY_IP]
ip route add default via 192.168.0.254
Utiliser "del" au lieu de "add" pour supprimer la route.

    network [IP_ADDRESS] [REVERSED_MASK] area [ZONE_OSPF]

        network 192.168.0.0 0.0.0.255 area 0

Interact with interfaces 
interface [INAME]
    ip ospf cost [COST]
    bandwith [BANDWITH_KBIT/S]

OSPFv6

Configuration 
ipv6 router ospf [PROCESS_ID] 
    router-id [UNIQUE_ID]
ipv6 router ospf 10 
    router-id 10.0.0.1


Ajouter une interface à annoncer
On n'annonce pas de "network" directement dans OSPF, on spécifie (au niveau des interface) qu'on veut qu'elles s'annoncent en tant que network connecté 
interface [INAME]
    ipv6 ospf [PROCESS_ID] area 0

    ! On peut changer le cout en faisant
    ipv6 ospf cost [COST]
interface Fa0/0/0
    ipv6 ospf 1O area 0
    ipv6 ospf cost 1

BGP

Configuration 
router bgp [AS_NUMBER]
    ! Annoncer les réseaux auxquels nous sommes connecté
    network [IP_ADDRESS] mask [MASK] 

    ! Au lieu d'annoncer chaque réseau auquel on est connecté à la main on peut taper 
    redistribute connected
    redistribute static

    ! Annonce nos voisins
    neighbor [IP_ADDRESS] remote-as [AS_OF_REMOTE_ROUTER]

    ! Redistribuer le BGP dans l'OSPF
    redistribute ospf [PROCESS_ID]

    ! Annonce l'AS de la confederation (l'"AS parent")
    bgp confederation identifier [AS]
    
    ! Annonce les autres AS dans la confederation 
    bgp confederation peers [AS_X] [AS_X] [AS...]

    ! Définir les clients du route reflector (si on en est un)
    neighbor [IP_ADDRESS] route-reflector-client
router bgp 65500
    network 10.0.0.0 mask 255.255.255.252
    redistribute connected
    redistribute static
    
    neighbor 10.0.0.1 remote-as 65601
    redistribute ospf 10

    bgp confederation identifier 65001
    bgp confederation peers 65002 65003 65004
    
    neighbor 10.0.0.3 route-reflector-client
Redistribuer l'OSPF dans le BGP 
router ospf [PROCESS_ID]
    redistribute bgp [AS] subnets
router ospf 10
    redistribute bgp 65500 subnets

VLAN

Create a vlan 
vlan [NUMBER]
    name [NAME_OF_THE_VLAN]
vlan 100
    name voice
Put an interface in a vlan 
interface [INAME]
    switchport mode access
    switchport access vlan [NUMBER]
interface Fa0/0/0
    switchport mode access
    switchport access vlan 100
Trunk encapsulation dot1q 
interface [INAME]
    switchport trunk encapsulation dot1q
    switchport mode trunk
    switchport trunk allowed vlan [VLAN_NUMBER1],[VLAN_NUMER2]
interface Fa0/0/0
    switchport trunk encapsulation dot1q
    switchport mode trunk
    switchport trunk allowed vlan 100,200

MAC

Change aging time 
mac address-table aging-time [VALUE (seconds)]
Create a static entry 
mac address-table static [MAC_ADDRESS xxxx.xxxx.xxxx] vlan 1 interface [INAME]
Clear the mac address table 
clear mac address-table dynamic

STP

Change the priority 
spanning-tree vlan 1 priority [PRIORITY]
Change duration of ... 
! Durée de Listening -> Learning & Learning -> Forwarding
spanning-tree vlan 1 forward-time [DURATION: 15]
! Récurrence d'envoi des trames
spanning-tree vlan 1 hello-time [DURATION: 2] 
! Durée après laquelle recalculer si aucun BPDU reçu
spanning-tree vlan 1 max-age [DURATION: 20]
Disable STP 
interface [INAME]
    spanning-tree portfast
Update the port 
interface [INAME]
    no negotiation auto
    speed 100
Clear the mac address table 
clear mac address-table dynamic

ACL

Create a standard ACL 
! Interdit le trafic provenant de .4
access-list [ACL_ID] deny host 192.168.1.4
! Autorise tout le reste
access-list [ACL_ID] permit any
Create an extended ACL 
! Filtre ICMP pour 192.168.1./24 quand dst => .23
access-list [ACL_ID] deny icmp 192.168.1.0 255.255.255.0 host 192.168.1.23

! Interdire accès au port 21
access-list [ACL_ID] deny tcp any host 192.168.1.23 eq 21

! Autorise le reste
access-list [ACL_ID] permit ip any any
ACL Standard: (0-100) Critère seulement sur l'@ip source
ACL Extended: (100-) Critère sur @ip_src, @ip_dst, port_src, port_dst, service

Enable ACL on interface 
interface [INAME]
ip access-group [ACL_ID] in

MPLS

Activer MPLS 
ip cef
mpls traffic-eng tunnels
Création de l'interface loopback 
interface [INAME]
    ip address [IP_ADDRESS] [MASK]
interface loopback0
    ip address 10.0.0.1 255.255.255.255
Activer MPLS sur les interfaces 
interface [INAME]
    mpls ip
    mpls traffic-eng tunnels
Ne tapez ces commandes que sur les interfaces concernés par MPLS

Configurer OSPF pour MPLS 
router ospf [PROCESS_ID]
    mpls traffic-eng router-id loopback0
    mpls traffic-eng area 0
    log-adjadency-changes
    
    ! On ajoute l'adresse de loopback
    network [LOOPBACK_IP] 0.0.0.255 area 0
Configurer un tunnel 
interface Tunnel1
    ip unnumbered Loopback0
    tunnel destination [DESTINATION_LOOPBACK_IP]
    tunnel mode mpls traffic-eng
    tunnel mpls traffic-eng autoroute announce
    tunnel mpls traffic-eng path-option 1 explicit name [NAME]
Configurer les IP de la route du tunnel 
ip explicit-path name [NAME]
    next-address [IP]
    next-address [IP]
    ! Jusqu'a l'ip non loopback du routeur vers qui y a le chemin
Tunnel unidirectionnel

Configurer un tunnel dynamique 
interface Tunnel1
    ip unnumbered loopback0
    tunnel destination [DESTINATION_LOOPBACK_IP]
    tunnel mode mpls traffic-eng
    tunnel mpls traffic-eng autoroute announce
    tunnel mpls traffic-eng priority 7 7
    tunnel mpls traffic-eng bandwith 300
    tunnel mpls traffic-eng path-option 1 dynamic

Configuration

Config hierarchy

system                -> configure hostname, domain name, DNS, user logins and permissions,
                         mappings between hostnames and addresses, and software PS                        
                        
protocols             -> configure BGP, IS-IS, LDP, MPLS, OSPF, RIP, and RSVP
interfaces            -> configure Fa/Gi interfaces
routing-options       -> configure static routes, autonomous system numbers, confederation
                         members, and global tracing
forwarding-options    -> configure forwarding options, including traffic sampling options.
groups                -> configure configuration groups 
firewall              -> configure filters that select packets based on their contents. 
security              -> configure IPsec services
policy-options        -> configure routing policies (filter / set properties in 
                         incoming/outgoing routes)
routing-instances     -> configure one or more routing instances
